## 0. 運用の全体像

| いつ | やること | 所要 |
| --- | --- | --- |
| **毎日（1回・5〜10分）** | 通報キュー・本人確認申請・問い合わせメールの確認 | 短い |
| **週次（1回・15〜30分）** | 重大通報の棚卸し、自動非表示になった投稿の妥当性チェック、指標確認 | 中 |
| **随時（通知が来たら）** | 未成年・犯罪等の重大案件は最優先で即対応 | 都度 |

> 基本方針：**軽微なものは自動処理に任せ、人が見るのは「重大・例外・グレー」だけ**。
> 迷ったら「ユーザーの安全 > 表現の自由 > 運営の手間」の順で判断する。

## 1. 本人確認（KYC）の手動レビュー運用

ユーザーが「設定 →本人確認 →身分証アップロード →申請」すると `verification_requests`
（status=`pending`）が作られます。**手順の詳細（承認/却下のSQL・画像削除）は
[SETUP.md 本人確認の運営手順](SETUP.md) に記載。** ここでは運用ルールを定めます。

### 確認頻度
- **1日1回**、`pending` を確認する（申請は多くないので基本これで十分）。
- 申請から**48時間以内**の承認/却下を目安（SLA）。遅れそうなら気にしすぎない ── β/初期は件数が少ない。

### 判断基準（承認の目安）
- ✅ **承認**：身分証の顔写真が鮮明／氏名・生年月日が読める／**18歳以上**が確認できる。
- ❌ **却下**：画像が不鮮明・一部隠れて年齢が確認できない／別人・加工が疑われる／マイナンバーの番号が写っている（※番号は取得しないので、番号を隠して再提出を案内）。
- ⚠️ **18歳未満と判明**：却下し、規約3章に基づきアカウントの利用停止を検討（本サービスは18歳以上限定）。

### 個人情報の取扱い（厳守）
- **確認が終わったら身分証画像を必ず削除**（R2 の `kyc/<user_id>/...`）。確認済みフラグだけ残す。
- マイナンバーカードは**番号を取得・保管しない**（顔写真面のみ）。
- この運用は[プライバシーポリシー §4](PRIVACY_DRAFT.md) と整合させること。

## 2. 通報対応の運用

通報は**自動処理が基本**です。一定数の通報が集まると DB トリガー
（`process_report_threshold`, SECURITY DEFINER）が対象を**自動で非表示**にします
（`content_flags`）。ブロックした相手の投稿も各画面で自動的に見えなくなります。
**運営が人手で見るのは以下だけ：**

### 随時（最優先・即対応）
次のカテゴリの通報は、閾値を待たず**発見/通知しだい最優先で対応**：
- **未成年に関する性的コンテンツ**（実在・創作を問わず）→ 即削除。必要に応じ関係機関へ通報。
- **犯罪・脅迫・自傷他害の予告** → 即対応。深刻なら警察等へ。
- **なりすまし・晒し（個人情報の無断公開）** → 確認のうえ削除。

### 週次（棚卸し）
- 自動非表示になった投稿を一覧で見て、**行き過ぎた非表示（濫用通報）がないか**を確認。
  誤りがあれば復帰させる（`content_flags` の該当行を解除）。
- 同一ユーザーへの通報が繰り返されていないか（規約違反の常習）を確認し、必要ならアカウント制限。

### 判断のグレーゾーン
- **露出の目安**：下着／下着と誤認される衣装がそのまま写る、性的部位を強調する構図は不可
  （衣装表現の範囲での露出はインナー等で処理されていることが前提）。→ 規約 第2章A に準拠。
- 迷ったら**まず非公開化してから本人に確認**（削除は最終手段）。

## 4. エスカレーション基準（人へ相談する線引き）

ワンオペでも、次は**自分だけで判断せず外部に相談**する：
- 法的リスクのある通報（名誉毀損・著作権侵害の削除要請、開示請求、警察・弁護士からの連絡）
  → **弁護士へ相談**。
- 大量の不正登録・スパム・攻撃 → 一時的に新規登録を絞る等の技術対応を検討。
- 決済（投げ銭）導入後の金銭トラブル → 決済事業者（Stripe）の規約・サポートに沿って対応。

## 5. 運用を支える実装（参考）

| 仕組み | 実装 | 人手の関与 |
| --- | --- | --- |
| 応募通知の自動生成 | `notify_on_application` トリガー | なし |
| 通報の自動非表示 | `process_report_threshold` トリガー＋`content_flags` | 例外のみ |
| 迷惑ユーザーの自動非表示 | ブロック＋`useModerationFilter`（home/search/market/qa/messages） | なし |
| 実績バッジの自動付与 | `refresh_meister_badge` トリガー | なし |
| 本人確認 | `verification_requests` ＋手動レビュー | あり（本書 §1） |

> 手動が捌けなくなったら：本人確認は eKYC（TRUSTDOCK 等）へ、NSFW は自動判定 API へ移行。
> いずれも「増えてから」で良い（[LAUNCH_PLAN.md](LAUNCH_PLAN.md) ローンチ後の項）。
