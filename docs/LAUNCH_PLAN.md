## ローンチまでの4ステップ

### STEP 1 — 法務・事業者情報の確定 　👤 オーナー中心
公開に不可欠。外部（弁護士）依存があるので**最優先で着手**。

- 🟡 👤 事業者情報の確定（運営者名・所在地・問い合わせ先）→ 規約/プライバシーの `〔　〕` を解消
      （運営者=Type&Co（屋号）・問い合わせ=16typeandco@gmail.com を確定し規約/プライバシー/アプリ内に反映済み。
      所在地は有償機能導入時に特商法表記として掲載。**残: 利用可能年齢の確定**（現行ドラフトは13歳の仮値））
- ⬜ 👤 弁護士による**利用規約・プライバシーポリシーのレビュー**
- 🟡 👤 Supabase / Cloudflare のリージョン・所在国を確認し、**越境移転の記載を確定**
      （Supabase=東京リージョン/米国法人・Cloudflare=エッジ型/米国法人 を確認し、規約/プライバシーに
      記載ドラフトを反映済み。**該当性・記載方法の最終確定は弁護士レビュー**）
- ⬜ 🛠 レビュー反映（文言修正を `TERMS_DRAFT.md` / `PRIVACY_DRAFT.md` と `TermsContent`/`PrivacyContent` に反映）
- ⏸ （有償機能を出す場合のみ）特定商取引法に基づく表記

### STEP 2 — 運用体制の確立（ワンオペ前提） 　👤 オーナー中心
「誰が・いつ・どう対応するか」を決める。仕組みは実装済みなので**運用ルールの確定**が主。

- ⬜ 👤 **本人確認の手動レビュー運用**（申請通知の受け取り方・確認頻度・承認/却下手順は [SETUP.md](SETUP.md) 済）
- ⬜ 👤 **通報対応の運用**（自動非表示は実装済み。例外・重大案件の確認フローを決める）
- ⬜ 👤 **お問い合わせ窓口**の用意（メール or フォーム）＋規約/プライバシーの窓口欄に反映
- ⬜ 🛠 （任意）本人確認申請が来たら運営に通知する仕組み（メール等）の追加

### STEP 3 — プロダクト最終確認 　🛠 開発中心
公開して恥ずかしくない状態にする。

- 🟡 🛠 **全機能の通し確認** — プロトタイプ全20画面のスモークは完了（クラッシュ/描画エラー 0）。
      **実機（本番ログイン状態）での通し確認はオーナー対応**（サンドボックスからは本番 DB に接続できないため）
- ✅ 🛠 **空データ時のUX** — ホーム/検索/メッセージ/おしらせ/知恵袋に空状態を追加（`EmptyState`）。
      ローンチ直後の空 DB でもブランクにならず、呼び水の導線を表示
- ⬜ 👤 **画像公開URL**（`NEXT_PUBLIC_R2_PUBLIC_URL`）の設定確認 — 未設定だと投稿画像が表示されない
      （コードは未設定でも安全にプレースホルダ表示。値の設定は環境変数側の作業）
- ⬜ 🛠 エラー時・通信不良時の表示確認
- ⬜ 👤 **初期コンテンツの投入**（呼び水になる併せ募集・イベント・作品など。過疎に見えない最低限）

### STEP 4 — 公開・集客 　👤 オーナー中心
