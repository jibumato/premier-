## 2. 特に確認してほしい論点（優先度順）

### 【高】① 個人データの越境移転（プライバシー §5）
- Supabase は**DBサーバーが東京（国内）だが、運営会社 Supabase Inc. は米国法人**。この構成が
  個人情報保護法28条の「**外国にある第三者への提供**」に該当するか。
- Cloudflare は**エッジ型で移転先国を特定できない**。この場合の「相当措置を継続的に確認する」
  という**代替記載で足りるか**（規則の要件を満たすか）。
- 現ドラフトの記載（§5.1）で過不足ないか。

### 【高】② 本人確認書類（身分証画像）の取扱い（プライバシー §4）
- 身分証画像を**確認後に速やかに削除**し、確認済みフラグのみ保持する運用で問題ないか。
- **マイナンバーは取得しない**（番号が写らない画像を求める）方針の妥当性。
- 目視確認のみ（eKYC未使用）で年齢確認（18歳以上）の根拠として十分か。

### 【高】③ 免責・責任範囲（規約 第7章 ほか）
- 併せ・撮影は**現実の対面活動**を伴う。当日の事故・ユーザー間トラブル・金銭トラブルについて
  運営が責任を負わない旨の免責が、**消費者契約法**に照らして有効な範囲か（過度な免責の無効リスク）。
- 「場を提供するだけ」という立て付けの妥当性。

### 【中】④ コスプレ特有の権利条項（規約 第1章・第5章）
- **「撮影の同意」と「掲載の同意」は別**とする条項の有効性。
- 写真について**カメラマンの著作権**と**被写体（レイヤー）の肖像権**が併存する前提の整理。
- 版権（原作）への配慮の求め方（二次創作・コスプレの位置づけ）。

### 【中】⑤ 通報・自動非表示・利用停止（規約 第6章）
- 通報が一定数集まると**自動で非表示**にする運用、重大違反での**アカウント停止**の
  規約上の根拠付けが十分か。濫用・誤判定時の異議申立て手当ての要否。

### 【中】⑥ 年齢制限の実効性（規約 第3章）
- **18歳以上限定**＋サインアップ時の自己申告チェック＋（任意の）本人確認、という
  多段構成で足りるか。18歳未満と判明時の利用停止対応の書き方。

### 【低・将来】⑦ 特定商取引法・資金決済法（投げ銭導入時）
- 現在は無償のため未対応。**投げ銭（Stripe Connect）導入時**に必要となる
  特商法表記・資金決済法（前払式支払手段該当性）の論点。※導入前に別途相談予定。
- 参考：[MONETIZATION.md](MONETIZATION.md)（Stripeが直接クリエイターへ支払う設計で前払式を回避する想定）。
