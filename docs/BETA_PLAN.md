# プルミエ！ クローズドβ 計画

> 一般公開の前に、少人数に配布して**不具合と使い勝手**を確認するための計画。
> ここを通すことで「本番ログイン状態での通し確認」（[LAUNCH_PLAN.md](LAUNCH_PLAN.md) STEP3）も
> 兼ねられます。

- **更新日**: 2026-07-11 ・ 版: beta v1.0
- **本番URL**: https://premier.sunny-rainy1115.workers.dev
- **窓口**: 16typeandco@gmail.com

---

## 1. 目的（βで確かめたいこと）

1. **致命的な不具合がないか**（登録・ログイン・投稿・メッセージが実データで通しで動くか）。
2. **使い勝手**（初見で迷わず使えるか、導線が分かるか）。
3. **空データの見え方**（過疎に見えないか、呼び水の導線が効くか）。
4. **本人確認・通報の運用が回るか**（申請→承認、通報→自動非表示を実際に体験）。

> βは「完璧さ」ではなく「**公開して恥ずかしくない最低ライン**」の確認が目的。

---

## 2. 規模・対象・期間

| 項目 | 目安 |

| 期間 | **1〜2週間** |

## 4. 集めるフィードバック項目

テスターに聞くと良い観点（アンケートにしても口頭でも可）：
- 詰まった・エラーが出た画面はどこか（できればスクショ）
- 登録〜最初の投稿まで、迷わずできたか（5段階）
- 「これが無いと使えない」と思った機能
- デザイン・文言で気になった所
- また使いたいと思うか（5段階）

---

## 5. β中に運営が見るチェックリスト

実機（本番ログイン）で自分でも一通り確認する ── STEP3「本番での通し確認」を兼ねる：

- [ ] 新規登録 → メール認証 → ログインが通る
- [ ] プロフィール編集・アバター/カバー画像アップロードが反映される
- [ ] 併せ募集の作成・一覧表示・詳細・応募が動く
- [ ] メッセージ送受信（Realtime 反映）・既読
- [ ] 投稿画像・アバターが**実際に表示される**（← `NEXT_PUBLIC_R2_PUBLIC_URL` 設定の確認）
- [ ] 本人確認を1件申請 → 運営側で承認 → バッジが付く（[OPERATIONS.md §1](OPERATIONS.md)）
- [ ] 通報を1件試す → 閾値で自動非表示になる（[OPERATIONS.md §2](OPERATIONS.md)）
- [ ] 空データの画面（初期状態）が呼び水表示になっている
- [ ] スマホ実機での表示崩れがない

> ⚠️ **画像URL（`NEXT_PUBLIC_R2_PUBLIC_URL`）が未設定だと投稿画像が表示されません**。
> β開始前に必ず設定を確認（[LAUNCH_PLAN.md](LAUNCH_PLAN.md) STEP3）。

## 6. β → 一般公開の判断基準（Exit条件）

次が満たせたら一般公開（STEP 4）へ進む：
- ✅ 致命的な不具合（登録できない・投稿が消える・画像が出ない等）が**ゼロ**。
- ✅ 「また使いたい」が過半数。
- ✅ 本人確認・通報の運用を**実際に1周**回せた。
- ✅ 規約・プライバシーの**弁護士レビュー完了**（[LAUNCH_PLAN.md](LAUNCH_PLAN.md) STEP1）。

見つかった不具合は優先度をつけて修正 → 直ったものからβに反映。
